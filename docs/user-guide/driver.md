# Руководство водителя

## Интерфейс

На смартфоне навигация **внизу экрана**: **Рейсы** и **Смена**.

## Рейсы

Отображаются только заявки:

- назначенные **вам** (`driverId` = ваш ID);
- с датой **сегодня** (по полю `scheduledAt`).

### Статусы

| Статус | Кнопка | Действие |
|--------|--------|----------|
| Назначена | Старт рейса | → В пути |
| В пути | Завершить | → Завершена |

## Закрытие смены

Раздел **Смена** или кнопка **Закрыть смену** на экране рейсов.

Заполните:

| Поле | Описание |
|------|----------|
| Автомобиль | ТС, на котором работали |
| Пробег нач./кон. | Одометр, км |
| Топливо нач./кон./заправка | Литры |

### Перерасход топлива

Система считает:

- пробег = конец − начало;
- норма = пробег / 100 × норма ТС (л/100 км);
- факт = топливо_нач + заправка − топливо_кон;
- перерасход % = (факт − норма) / норма × 100.

Если **> 5%** и комментарий пустой — появится предупреждение, смена не закроется. Введите причину (пробки, холостой ход и т.д.) и отправьте снова.

## API (мобильный браузер)

```bash
# Старт рейса
curl -X POST http://SERVER:8080/api/orders/status \
  -H "Authorization: Bearer TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"orderId":"UUID","status":"in_progress"}'

# Закрытие смены
curl -X POST http://SERVER:8080/api/shift/close \
  -H "Authorization: Bearer TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "vehicleId":"UUID",
    "startOdometer": 1000,
    "endOdometer": 1045,
    "fuelStart": 40,
    "fuelEnd": 8,
    "fuelRefilled": 15,
    "comment": ""
  }'
```
