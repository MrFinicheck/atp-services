# Руководство диспетчера

## Зона ответственности

Приём заявок на перевозку, назначение ресурсов (ТС + водитель), мониторинг загрузки парка.

## Создание заявки — пошагово

1. **Создать** в верхней панели.
2. **Клиент** — из справочника (добавление клиентов: POST `/api/clients` или через админа).
3. **Тариф** — влияет на расчёт цены.
4. **Маршрут** — текстовые адреса «откуда» / «куда», километраж.
5. **Простой** — часы ожидания на точке.
6. **Срочная** — да/нет (увеличивает цену).
7. **Автомобиль** — только активные.
8. **Водитель** — из списка с ролью driver.
9. Дождитесь строки **Расчёт: … ₽**.
10. **Создать заявку**.

Заявка создаётся со статусом **Назначена** (`assigned`).

## График парка

Используйте **Парк** перед назначением новой машины:

- Свободная полоса — можно планировать рейс на этот госномер.
- Несколько блоков — несколько заявок на одно ТС в один день (последовательно).

## Клиенты

Диспетчер может **сохранять** клиентов (`POST /api/clients`), просматривать список.

Пример тела:

```json
{
  "name": "ООО Ромашка",
  "phone": "+74951234567",
  "debtLimit": 100000
}
```

## Чего диспетчер не делает

- Не редактирует тарифы и автопарк (только admin).
- Не закрывает смену водителя.
- Не видит отчёт по рейтингу водителей в UI (только admin).
