# Руководство администратора

## Обязанности в системе

Администратор управляет справочниками, пользователями, контролирует отчётность и имеет полный доступ к операциям диспетчера.

## Ежедневные задачи

1. **Обзор** — проверка KPI и перерасходов топлива.
2. **Рейсы** — контроль статусов заявок.
3. **Парк** — загрузка автопарка на день.
4. **Отчёты** — рейтинг водителей, выручка за месяц.

## Справочники

### Тарифы

Раздел **Тарифы** → форма внизу:

- Название, подача (₽), ₽/км, ₽/ч простоя.
- Коэффициент срочности по умолчанию 1.5 (задаётся в коде при сохранении через UI).

### Автопарк

Раздел **ТС**:

- Госномер, модель, норма расхода (л/100 км).
- Неактивные ТС не попадают в выбор при создании заявки.

### Пользователи (раздел «Сотрудники»)

В интерфейсе: **Сотрудники** → форма «Новый сотрудник».

Можно создать:
- **Водителя** — логин, пароль, ФИО, телефон;
- **Диспетчера**;
- **Администратора**.

Список всех учётных записей отображается выше формы (без паролей). У каждой строки (кроме своей) — кнопка удаления с подтверждением.

Ограничения при удалении:
- нельзя удалить свою учётную запись;
- нельзя удалить последнего активного администратора.

Через API — `POST /api/users` (создание), `DELETE /api/users?id={uuid}` (удаление):

```json
{
  "user": {
    "login": "driver3",
    "role": "driver",
    "firstName": "Иван",
    "lastName": "Петров",
    "phone": "+79001234567",
    "active": true
  },
  "password": "secret123"
}
```

Удаление:

```http
DELETE /api/users?id=uuid-пользователя
Authorization: Bearer <token>
```

## Журнал аудита

`GET /api/audit` — действия: создание заявок, смена статуса, закрытие смены и т.д.

## Ограничения

- Нельзя обойти блокировку смены при перерасходе топлива без комментария (бизнес-правило 5%).
- Пароли хранятся как bcrypt-хеш; восстановление — только сброс через создание нового пользователя или правку БД.
