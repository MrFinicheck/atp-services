# Обоснование архитектурных решений (ADR)

## ADR-001: Go + LevelDB вместо PostgreSQL

**Контекст:** дипломный проект для малого АТП, один офис, без выделенного DBA.

**Решение:** встроенное хранилище LevelDB в каталоге данных.

**Плюсы:**
- Нулевая настройка СУБД, перенос «папкой» `data/`.
- Быстрый старт для Wails desktop.
- Достаточный объём для сотен заявок и десятков ТС.

**Минусы:**
- Нет ad-hoc SQL-отчётов (компенсируется API отчётов).
- Один писатель на каталог (flock).

**Альтернатива:** SQLite — рассматривалась; LevelDB выбран за простоту key-value модели под JSON-сущности.

## ADR-002: Wails v2 + общий REST для веб-режима

**Контекст:** нужен и desktop, и доступ с телефона водителя по Wi‑Fi.

**Решение:**
- Бизнес-логика в `internal/app` + `internal/services`.
- Wails (`app.go`) и HTTP (`internal/api`) — тонкие адаптеры.
- Frontend один: `api/client.ts` переключает Wails/HTTP.

**Плюсы:** DRY, одна кодовая база UI, демонстрация на защите в двух режимах.

---

## ADR-003: Слой ports (DIP)

**Контекст:** требования к SOLID в дипломе.

**Решение:** интерфейсы репозиториев и сервисов в `internal/ports`, реализация — `store.Store`.

**Плюсы:** сервисы тестируются с temp LevelDB; замена хранилища без переписывания логики.

---

## ADR-004: RBAC в Application, не в UI

**Решение:** проверки ролей в `internal/app/app.go` (`requireUser`, `errAccessDenied`).

**Плюсы:** нельзя обойти ограничения прямым вызовом API; UI только скрывает кнопки.

## ADR-005: Пустой slice для ScheduleToday

**Проблема:** Go `nil` slice → JSON `null` → падение UI `orders.length`.

**Решение:**
- Backend: `Orders: []models.Order{}` в `ScheduleToday`.
- Frontend: `normalizeSchedule`, `item.orders ?? []`.

---

## ADR-006: Контроль топлива 5%

**Решение:** `fuelOverrunTolerance = 0.05` в `WaybillService`; без комментария `RequireComment: true`.

**Обоснование:** типовое требование учёта ГСМ в АТП; снижает мошенничество и ошибки заправки.

---

## ADR-007: Сессии в LevelDB, не JWT

**Решение:** случайный token 32 байта, запись `session:{token}`, TTL 24ч.

**Плюсы:** простой logout (удаление ключа), не нужен secret для подписи JWT.

**Минусы:** не масштабируется на кластер без sticky session — для диплома приемлемо.

---

## ADR-008: UI TransitOS (dock + dark terminal)

**Решение:** горизонтальный dock вместо sidebar, bento dashboard, gantt-график.

**Обоснование:** отличие от «типового админ-шаблона»; акцент на диспетчерский терминал.
